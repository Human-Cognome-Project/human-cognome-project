# Interfaces, Theory of Mind, and the Translation Bridge

This page covers the architecture's central *epistemic* commitment: the discipline of holding
every interface **as an interface**, never presuming it captures complete reality. This single
discipline is what both **forbids** a core failure mode and **enables** NAPIER's most ambitious
capability — cross-interface (including cross-species) translation.

Sources: claims 244 (presumptive semantic imperative), 245 (precept-level uncertainty),
246 (interface-agnostic capability), 247 (translation bridge), 82/83 (multi-ToM), 87/94 (PBD),
261/262 (construction vs usage).

---

## The presumptive semantic imperative

> *A **presumptive semantic imperative** is the unwarranted assumption that one's own semantic
> interface — category set, perceptual receptors, language frame, level of detail — captures
> **complete** reality: mistaking the interface for the territory.* — claim 244

The illustration Patrick uses is color perception. Humans presume "color" is complete, but
3 receptors give us ~1M colors, while a mantis shrimp has 16 receptors plus polarized-light
sensing, a pigeon has 5 (including UV), a reindeer sees UV, a butterfly up to 15. Each inhabits
a **different reality off the same planet.** What an interface does not capture is **not
absent** — it is *always there*, invisible and unmodeled.

This is a **unifying failure mode the whole architecture is built to resist.** Its throughlines:

- **null-vs-zero** (claim 17) — uncaptured reality is −0 (there but unknown), not proven-absent.
  Presuming completeness illegitimately collapses −0 into false certainty.
- **PBD / multi-ToM** (claims 82–83, 87) — every perspective is a bounded interface; reality is
  the *intersection across* interfaces, never any single one.
- **Conceptual hub** (claim 240) — the concept layer must hover between expression modes so it
  is never welded to one limited interface. Binding concept to English-text alone would **be**
  this failure (see [linguistic-vs-conceptual.md](linguistic-vs-conceptual.md)).
- **NSM English-bias** (claim 200) — treating English-glossed NSM as the territory rather than
  an on-ramp is this presumption.
- **LLM critique** — an LLM presumes its training-distribution interface *is* reality; other
  minds' receptors are the always-there interfaces it structurally cannot see.

**The discipline:** hold the interface *as* an interface — warranted, bounded, never presumed
complete (warrant/confidence, claims 22, 122). NAPIER's job is to model reality without
presuming its current interface is the whole of it.

## Perspective-Based Dimensions (PBD)

**PBD = Perspective Based Dimension** (claim 94, Patrick's coined term, load-bearing for the
concept space). Each PBD is a frame in which factors orient. Multiple PBDs **intersect** rather
than conflict.

> *Cross-frame "conflict" is a flattening artifact, not real disagreement. Real structure is
> intersection geometry — orthogonal PBDs at an intersection point appear as conflict only when
> collapsed to one dimension.* — claim 87

Making progress on a hard disagreement often means making the *dimensional structure visible*,
so observers see intersection geometry instead of a flat clash. ("Same planet, different
realities" is PBD orthogonality, literally.)

## Multi-ToM: four-plus concurrent constructs

> *Bidirectional dialogue runs 4+ concurrent Theory-of-Mind constructs per participant.* — claim 82

The four base constructs: self-as-self, self-as-modeled-by-other, other-as-other,
other-as-modeled-by-self. Every factor is ToM-relative — the word "I" is always a localized
speaker-deictic. The direct/indirect axis (2nd vs 3rd person) routes to the appropriate ToM
construct. **Each ToM construct is its own PBD**, and recursive depth is available (ToM layering
grows specious with depth — claim 83).

All expression exists to convey Theory of Mind to a recipient (claim 200) — so multi-ToM is not
a dialogue feature bolted on; it is the substrate of communication itself.

## The capability flip-side: interface-agnostic

The same property that forbids the arrogant presumption *enables* a capability (claim 246):

> *NAPIER is not bound to the human interface. There is no reason it could not "see" the world
> as a mantis shrimp if the wavelengths each cone receives were isolated and supplied — that is
> just another interface to map onto the conceptual hub.*

Because NAPIER never presumes its current interface is reality, it can ingest and map
**arbitrary** interfaces: non-human perception, other modalities, other languages — all the
same operation.

**A key nuance on uncertainty (claims 245/246):** the cost of crossing into another interface
depends on whether you can *sample* it.

- When the raw signal **can be supplied** (sampled directly), the problem shifts from
  *estimating-an-alien-by-analogy* toward *directly mapping a new interface* — you learn the
  real interface instead of reconstructing it from your own. Precept-level uncertainty drops
  sharply.
- When a side can **only be estimated** by analogy, the uncertainty is at the **precept /
  framework level**, not merely the data level: your very tools for mapping the other interface
  are made of your own limited material, so the reconstruction can fail at the *foundation*,
  not just the edges. (This is the *Linguistic Archaeology* move — see
  [../01-foundations/](../01-foundations/) article 03.)

So precept-uncertainty is **highest** for interfaces you cannot sample, **lowest** for those
whose raw signal you can feed in.

## The translation bridge

> *If NAPIER can sufficiently model **both sides** of meaning — each party's interface mapped
> onto the shared conceptual substrate — then translation is possible, because translation **is**
> routing meaning through the shared, language-neutral concept layer between two interfaces.* — claim 247

This is fully general: human↔animal, human↔human across languages, and any two mappable
interfaces are the same operation. Cross-species is just the most dramatic instance.

The mechanism is **construction-vs-usage separation** (claims 261/262):

- **Linguistic rules (grammar / PoS) are construction blueprints** — meaning-agnostic; they
  only enforce that pieces are in the right place and cohere.
- **The usage plan (the *why* / the semantics) is the invariant.** Different languages are
  different architectural schools: each produces structurally very different but equally valid
  constructions. Validity is "stands and coheres," not "matches a canonical form" — so there is
  no single correct construction.
- **To translate is to re-construct the same usage plan in a different school's blueprint.**
  This is why wildly different grammars are all legitimate *and* mutually translatable.

The shared usage plan is the conceptual hub (claim 240) that the bridge routes through. The
bridge is only as honest as its **weaker-modeled side** — the humility (244/245) and the reach
(246) are the same property.

## Honesty note

The cross-species translation bridge is a **vision-level destination** (claim 247), not a
shipped capability. It is the endpoint the interface arc points at. Phase 4 multi-modality (see
[../06-status/../ROADMAP.md](../../ROADMAP.md)) is the same mechanism extended to audio/visual
expression. What is *built today* is the single-language (English text) case; the generality is
architectural intent, documented here as intent.
