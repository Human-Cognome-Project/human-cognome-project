# What Is NAPIER?

A one-page orientation. If you read nothing else first, read this.

---

## Digital Intelligence, not AI

> *HCP is **Digital Intelligence (DI)**, not AI.* — claim 93

The Human Cognome Project builds cognitive infrastructure grounded in the physics of thought. It is
**not** NLP, **not** an alternative LLM, **not** a statistical model. The core difference:

- An LLM **simulates** language output (a *Phänomenmodell*) without executing language logic.
- NAPIER **executes** the logic — it performs, transparently and composably, the database operations
  that cognition reduces to (a *Gedankenmodell*). See
  [../01-foundations/](../01-foundations/).

**The data IS the model** — structured semantic artifacts that compose and reason, not weights that
approximate. Text is the proving ground (abundant, verifiable), not the goal.

**NAPIER** is the name of the inference model (claim 202).

---

## The one realization everything follows from

> Cognition reduces to **database functions** at the core.

Categorization is a classification query; combination is a join / parameter-binding; grammar is the
function-call API. LLMs are *already* doing db functions — opaquely. NAPIER does the same operations
in the open. Once you see this, the rest of the architecture is logical followthrough. This is **the
keystone** — [../02-architecture/keystone-db-functions.md](../02-architecture/keystone-db-functions.md).

## What it's built on

- **Knowledge lives in the connections**, not isolated facts (claim 140). Intelligence is the
  **quality of the data × the ability to traverse it** (claim 255).
- **Two concurrent spaces**: a 3D Newtonian physics space that grounds perception (and *is*
  imagination), and a cognitive concept space (claims 19/253/254).
- **A fixed set of concept primes** (the 65 NSM primes) that combine into molecules — a literal
  periodic table for concepts (claims 219/220).
- **An engine with two sides**: a GPU **resolution furnace** that surfaces deltas at speed, and a
  CPU-orchestrated **inference walk** that interprets them (claims 57/281).
- **Honest epistemics**: hold every interface *as* an interface, never presumed complete (claim 244);
  distinguish *unknown* (−0) from *proven-false* (+0) (claim 17); fill gaps only with warrant, never
  confabulation.

## What's actually built (and what isn't)

NAPIER is **real, running software** — an O3DE/PhysX 5 engine that ingests text and reproduces it at
>98% accuracy, on a ~1.494M-entry English substrate. But significant parts are **intended, not yet
built**, and several areas are **explicitly deferred**. The docs flag this everywhere, and collect it
on one page:

- **Current status:** [../06-status/status.md](../06-status/status.md)
- **What's deferred / in-flux / planned:** [../06-status/deferred-and-open.md](../06-status/deferred-and-open.md)

Read those two alongside any architecture page so you never mistake intent for done-state.

## The proof target

> Grade-school reading comprehension — the lower grade the better (claim 23).

The Phase 1 proof is **traceable inference on bounded data, not scale.** A kindergarten primer
("Bob's Pets") is the concrete test. If the substrate can demonstrably *understand* at that level —
traceably, reproducibly — the paradigm is proven. See [../../ROADMAP.md](../../ROADMAP.md).
